# content/

Draft X threads, blog posts, and screencast scripts. Source material lives in [`../BUILD_LOG.md`](../BUILD_LOG.md). Polish happens here.

Structure:
- `threads/` — X thread drafts, one file per thread, numbered. Each tweet on its own line, separated by blank lines. Image / screenshot slots marked with `[IMG: description]`.
- `posts/` — long-form (dev.to, Substack, Medium). One file per post.
- `scripts/` — screencast scripts. What I say, what I'm clicking, the order.

Workflow: build something interesting. Update `BUILD_LOG.md`. When the build log entry feels like enough material for a thread or post, copy it here and remix into the platform's voice. X likes punchier openings and image-heavy threads. Long-form blogs can carry the full argument with diagrams.
